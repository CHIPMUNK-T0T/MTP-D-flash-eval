# Technical Document: KV Cache Memory Optimization

## Part 1: Technical Analysis

Large language models (LLMs) have fundamentally transformed how we process and generate natural language at scale. These models learn statistical patterns from vast corpora encompassing web pages, books, academic papers, and code repositories. The transformer architecture introduced in 2017 remains the dominant paradigm, enabling models to attend to distant token dependencies efficiently through multi-head attention.

The key-value (KV) cache is the central optimization enabling efficient autoregressive inference. During generation, each forward pass computes query, key, and value projections for the current token. The key and value tensors are stored in the KV cache so subsequent tokens can reuse them without recomputation. Without this cache, inference would require O(T^2) computation for a sequence of T tokens; with the cache, it is O(T) amortized.

Memory consumption is the primary constraint in deploying large models on consumer GPU hardware. For a model with L layers, H_kv key-value heads, and head dimension D, each token occupies 2 * L * H_kv * D * bytes_per_element bytes in the KV cache. For the Qwen3.5-4B architecture with 36 layers, 8 KV heads, and D=128 using BF16 (2 bytes), this amounts to 2 * 36 * 8 * 128 * 2 = 147,456 bytes per token, before accounting for block alignment overhead in block-based allocators.

FP8 quantization reduces each element from 16 bits to 8 bits, halving the KV cache footprint. On an RTX 4070 with 12 GB VRAM, after loading the FP8 model weights and reserving activation memory, approximately 3 GB remains for KV storage. With BF16 KV, this accommodates roughly 90,000 token slots. With FP8 KV, the same 3 GB accommodates approximately 175,000 token slots, a 1.94x improvement that directly translates to longer supported sequences.

The vLLM serving framework implements PagedAttention, which divides the KV cache into fixed-size blocks (typically 16 tokens each). When a new request arrives, vLLM allocates the minimum number of blocks required for the prompt, then allocates additional blocks incrementally as the model generates output tokens. This eliminates fragmentation and allows different requests to share physical KV blocks for common prefixes through prefix caching.

Serving reliability depends on ensuring the KV pool can accommodate at least one full-length request. vLLM validates this at startup: if the number of available KV blocks times the block size is less than max_model_len, it raises a ValueError and refuses to start. This is the mechanism by which BF16 KV fails at 96K max_model_len (89,600 available slots < 98,304 required) while FP8 KV succeeds (174,000 available slots > 98,304 required).

Benchmarking inference throughput requires careful attention to warmup effects. Modern GPU frameworks use just-in-time compilation via Triton for custom kernels, which causes the first inference call to be substantially slower than subsequent calls. Accurate throughput measurements must discard at least one warmup request before recording latency and token-per-second figures.

Time-to-first-token (TTFT) is a user-facing latency metric distinct from generation throughput. TTFT measures the delay from request submission until the first output token is streamed back to the client. It is dominated by the prefill phase, which processes all input tokens in a single forward pass. Longer input sequences increase TTFT proportionally, making it sensitive to context length in a way that generation throughput is not.

Speculative decoding is a complementary technique that uses a smaller draft model to predict multiple tokens ahead, with the main model verifying them in a single forward pass. When the acceptance rate is high, this can substantially increase effective generation throughput. However, speculative decoding and KV cache quantization are independent optimizations that can be combined for multiplicative benefit.

The tradeoff between model quantization and KV cache quantization differs importantly. Model weight quantization reduces the memory footprint of the neural network parameters and can accelerate matrix multiplications on hardware with dedicated low-precision compute units. KV cache quantization reduces only the memory used to store attention states during inference, without changing the weight layout. Both can be applied simultaneously.
